CS661 Assignment 1 - 2D Isocontour Extraction

Folder Structure:

Assignment_1/
│
├── simul.py
├── README.txt
└── Data/
    └── Isabel_2D.vti

Ensure that the dataset file `Isabel_2D.vti` is placed inside the `Data` folder located in the same directory as `simul.py`.

Running the Program:

The script takes the isovalue as a command-line argument.

Syntax:

python simul.py <isovalue>

Examples:

python simul.py 0

python simul.py -600

python simul.py 300

Input:

<isovalue> : Floating-point value specifying the contour value to be extracted.

Output:

The program generates a VTK PolyData (`.vtp`) file containing the extracted isocontour.

Output file name format:

contour_<isovalue>.vtp

Examples:

contour_0.vtp

contour_-600.vtp

contour_300.vtp

Visualization:

The generated `.vtp` file can be opened directly in ParaView for visualization of the extracted isocontour.

Special Cases:

1. Isovalue Outside Scalar Range

If the specified isovalue lies outside the scalar range of the dataset, the program displays a warning message. Since no contour can exist outside the scalar range, the generated output will contain no contour segments.

2. Ambiguous Cell (Four Intersections)

In certain cells, the isocontour may intersect all four edges of the cell, producing four intersection points. As specified in the assignment, ambiguity resolution techniques such as the Asymptotic Decider are not implemented.

The cell edges are traversed in the following counterclockwise order:

Bottom → Right → Top → Left

The intersection points are stored in this order, and two contour segments are generated by connecting consecutive pairs of intersections:

(Bottom, Right)

(Top, Left)

This follows the prescribed edge traversal order while avoiding ambiguity resolution.
